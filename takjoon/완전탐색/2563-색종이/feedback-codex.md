# 2563 색종이 피드백

## 통과 여부

예제 입력 기준으로 정답 `260`이 출력되는 것을 확인했다.
다만 실제 제출의 모든 숨은 케이스 통과 여부는 백준 채점 결과를 기준으로 판단해야 한다.

## 잘한 점

- 문제의 좌표 범위가 `100 x 100`으로 작다는 점을 잘 활용했다.
- 색종이가 덮는 칸을 2차원 배열에 직접 `1`로 표시해서, 겹치는 영역을 자연스럽게 한 번만 세도록 만들었다.
- 복잡한 좌표 계산이나 포함-배제 대신 배열 표시 방식으로 풀어서 실수 가능성이 낮다.
- `sum(matrix)`가 아니라 각 행의 합을 따로 더하는 방식으로 2차원 리스트의 합계를 올바르게 계산했다.

## 개선하면 좋은 점

- 입력 예시를 파일 상단의 주석으로 남기는 것은 학습 중에는 좋지만, 제출 전에는 제거해도 된다.
- `paint` 함수 안의 `if matrix[i][j] == 0` 조건은 없어도 된다. 이미 칠해진 칸에 다시 `1`을 넣어도 결과가 같기 때문이다.
- 정답 계산 부분은 현재처럼 `answer`에 행별 합을 누적해도 좋다. 익숙해지면 `sum(sum(row) for row in matrix)`처럼 더 짧게 표현할 수도 있다.
- `paint` 함수가 전역 변수 `matrix`를 직접 바꾸고 있으므로, 더 큰 프로그램에서는 의존성이 숨어 보일 수 있다. 이 문제에서는 괜찮지만, 나중에는 필요한 값을 함수 인자로 넘기는 방식도 연습해보면 좋다.
- 변수명 `number_of_paper`도 의미는 통하지만, 여러 장이라는 뜻을 더 살리려면 `paper_count`처럼 짧게 써도 읽기 좋다.

## 핵심 아이디어

색종이 한 장은 항상 `10 x 10` 칸을 덮는다.
각 색종이의 왼쪽 아래 좌표를 기준으로 `x`부터 `x + 9`, `y`부터 `y + 9`까지의 칸을 모두 `1`로 표시한다.
이미 칠해진 칸을 다시 칠해도 값은 그대로 `1`이므로, 겹치는 영역은 중복해서 세지 않는다.
마지막에는 100개의 행을 돌면서 각 행에 있는 `1`의 개수를 더하면 전체 검은 영역의 넓이가 된다.

## Generator 메모

`sum(row) for row in matrix`는 리스트가 아니라 generator expression이다.
generator는 값을 한 번에 모두 만들어 저장하지 않고, 필요할 때 하나씩 계산해서 꺼내는 객체다.

예를 들어 `sum(sum(row) for row in matrix)`라고 쓰면, 바깥쪽 `sum`이 각 행의 합을 하나씩 받아서 전체 합계를 만든다.
이 흐름은 아래 코드와 같은 의미다.

```python
answer = 0
for row in matrix:
    answer += sum(row)
```

반면 `[sum(row) for row in matrix]`처럼 대괄호를 쓰면 각 행의 합을 담은 리스트를 먼저 만든다.
이번 문제는 도화지가 `100 x 100`이라 차이가 크지는 않지만, generator 표현식은 불필요한 리스트를 만들지 않는다는 장점이 있다.

## 다음 문제에 적용할 교훈

좌표 범위가 작고 겹치는 영역을 세야 하는 문제에서는, 수식으로 겹침을 직접 계산하기보다 배열에 표시한 뒤 세는 방식이 더 안전할 때가 많다.
또한 2차원 리스트 전체의 합을 구할 때는 리스트 자체를 바로 `sum`하지 말고, 행 단위로 합을 구한 뒤 다시 더해야 한다.
