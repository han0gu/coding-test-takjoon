# Codex 피드백

## 현재 풀이에 대한 평가

현재 `main.py`의 방향은 좋습니다.

- 문제 조건: `0 <= N <= 2,147,483,647`
- 목표: `N`이 서로 다른 3의 거듭제곱들의 합으로 표현될 수 있는지 판별
- 중요한 조건: 3의 거듭제곱을 하나 이상 사용해야 하고, 같은 거듭제곱은 두 번 이상 사용할 수 없음

처음에는 DFS로 각 거듭제곱을 선택할지 말지 탐색하는 방식이었지만, 지금은 `N`을 3진수 관점에서 확인하는 방식으로 바뀌었습니다.
이 문제에서는 이 방향이 더 단순하고 실수하기 어렵습니다.

현재 풀이의 핵심은 다음입니다.

```python
while n > 0:
    tenery_digits += str(n % 3)
    n //= 3

print('NO' if tenery_digits.find('2') > -1 else 'YES')
```

`n % 3`으로 3진수의 각 자리 숫자를 하나씩 확인하고, 그중 `2`가 있으면 같은 3의 거듭제곱을 두 번 써야 하므로 `NO`로 판단합니다.

## 왜 DFS를 하지 않아도 될까?

DFS로 푼다면 다음처럼 생각하게 됩니다.

- `3^0`을 쓸까?
- `3^1`을 쓸까?
- `3^2`를 쓸까?
- ...

즉, 각 3의 거듭제곱을 선택하거나 선택하지 않는 조합을 직접 훑는 방식입니다.

하지만 어떤 수를 3진수로 표현하면, 각 자리 숫자가 이미 해당 거듭제곱을 몇 번 쓰는지를 나타냅니다.

예를 들어:

```text
109 = 3^4 + 3^3 + 3^0
```

이를 3진수로 보면:

```text
109 = 11001(3)
```

각 자리는 다음 뜻입니다.

| 3진수 자리 | 의미 | 사용 여부 |
|---|---|---|
| `3^4` 자리의 `1` | `3^4`를 1번 사용 | 사용 |
| `3^3` 자리의 `1` | `3^3`을 1번 사용 | 사용 |
| `3^2` 자리의 `0` | `3^2`를 0번 사용 | 사용 안 함 |
| `3^1` 자리의 `0` | `3^1`을 0번 사용 | 사용 안 함 |
| `3^0` 자리의 `1` | `3^0`을 1번 사용 | 사용 |

그래서 조합을 따로 찾지 않아도, 3진수 표현 자체를 보면 어떤 거듭제곱을 쓰는지 알 수 있습니다.

## 왜 나머지가 3진수의 한 자리일까?

어떤 수 `N`의 3진수 표현은 다음처럼 쓸 수 있습니다.

```text
N = a_k * 3^k + ... + a_2 * 3^2 + a_1 * 3 + a_0
```

여기서 각 `a_i`는 3진수의 한 자리이므로 `0`, `1`, `2` 중 하나입니다.

이 식을 3으로 나눈 나머지를 보면:

```text
a_k * 3^k, ..., a_2 * 3^2, a_1 * 3
```

이 항들은 모두 3의 배수입니다.
따라서 3으로 나눈 나머지에는 가장 오른쪽 자리인 `a_0`만 남습니다.

즉:

```text
N % 3 = a_0
```

그리고 `N // 3`을 하면 가장 오른쪽 자리 `a_0`을 제거하고, 나머지 자리들을 한 칸씩 오른쪽으로 민 것과 같습니다.
그래서 `n % 3`, `n //= 3`을 반복하면 3진수의 각 자리를 오른쪽부터 하나씩 확인할 수 있습니다.

## `2`가 나오면 왜 안 될까?

3진수의 각 자리 숫자는 해당 거듭제곱을 몇 번 쓰는지를 뜻합니다.

| 나머지 | 의미 | 가능 여부 |
|---|---|---|
| `0` | 그 자리의 `3^k`를 사용하지 않음 | 가능 |
| `1` | 그 자리의 `3^k`를 한 번 사용 | 가능 |
| `2` | 그 자리의 `3^k`를 두 번 사용해야 함 | 불가능 |

문제에서는 같은 3의 거듭제곱을 겹치지 않고 한 번씩만 사용할 수 있습니다.
따라서 한 자리라도 `2`가 나오면 삼삼한 수가 아닙니다.

예를 들어 `7`은:

```text
7 = 21(3)
```

이므로 `3^1`을 2번 써야 합니다.

```text
7 = 3 + 3 + 1
```

하지만 `3^1`을 두 번 쓰는 것은 문제 조건에 어긋나므로 `NO`입니다.

## 틀렸던 이유

### 1. 디버깅 출력이 남아 있었음

중간에 다음 출력이 남아 있으면 제출 결과가 틀립니다.

```python
print("answer", answer)
```

문제는 출력이 정확히 `YES` 또는 `NO` 한 줄이어야 합니다.
디버깅 출력이 있으면 로직이 맞아도 출력 형식이 달라져서 오답 처리됩니다.

### 2. `N = 0` 예외 처리가 필요했음

문제 입력은 `0`도 가능합니다.
하지만 문제에서 삼삼한 수는 3의 거듭제곱인 수가 반드시 하나 이상 포함되어야 한다고 했습니다.

따라서 `0`은 아무 3의 거듭제곱도 사용하지 않으므로 `NO`가 맞습니다.

이 처리가 없으면:

```python
n = 0
tenery_digits = ''

while n > 0:
    ...

print('NO' if tenery_digits.find('2') > -1 else 'YES')
```

`while`문이 한 번도 실행되지 않고, 빈 문자열에는 `'2'`가 없으므로 `YES`가 출력됩니다.
그래서 `N == 0`은 따로 처리해야 합니다.

```python
if n == 0:
    print("NO")
```

## 개선하면 좋은 점

### 1. 변수명 오타 수정

현재 변수명은 `tenery_digits`입니다.
의도는 3진수 자리들이므로 영어로는 `ternary_digits`가 더 정확합니다.

```python
ternary_digits = ''
```

다만 이 문제에서는 실제 3진수 문자열 전체가 필요한 것은 아니고, `2`가 있는지만 보면 됩니다.

### 2. 문자열을 만들지 않고 바로 판별하기

현재 방식도 정답을 판별할 수 있습니다.
하지만 문자열을 끝까지 만든 다음 `'2'`를 찾기보다, 나머지가 `2`인 순간 바로 `NO`로 판단할 수도 있습니다.

개선 예시는 다음과 같습니다.

```python
n = int(input())

if n == 0:
    print("NO")
else:
    is_sam = True

    while n > 0:
        if n % 3 == 2:
            is_sam = False
            break
        n //= 3

    print("YES" if is_sam else "NO")
```

이 코드는 3진수 문자열을 저장하지 않고, 각 자리 숫자를 확인하면서 불가능한 경우 바로 멈춥니다.

## 복잡도

`N`을 3으로 계속 나누므로 반복 횟수는 3진수 자리 수만큼입니다.

- 시간 복잡도: `O(log N)`
- 공간 복잡도:
  - 현재 문자열 방식: `O(log N)`
  - 바로 판별하는 방식: `O(1)`

`N`이 최대 `2,147,483,647`이어도 3진수 자리 수는 많지 않으므로 시간은 충분합니다.
