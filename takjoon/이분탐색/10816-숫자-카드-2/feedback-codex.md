# 10816 숫자 카드 2 피드백

## 문제 조건 요약

- 상근이가 가진 숫자 카드 `N`개와, 확인해야 할 숫자 `M`개가 주어진다.
- `N, M`은 각각 최대 `500,000`이다.
- 각 확인 숫자에 대해 상근이가 해당 숫자 카드를 몇 장 가지고 있는지 공백으로 출력한다.
- 숫자 카드에는 같은 수가 여러 번 나올 수 있다.

## 처음 풀이에서 확인한 점

`Counter(n_list)`를 사용해 보유 카드 개수를 세는 방향은 좋았다.

다만 아래처럼 문자열에 계속 더하는 방식은 문제가 있었다.

```python
answer += str(counter[m]) if counter[m] else '0'
```

문자열은 한 번 만들어지면 직접 수정되지 않기 때문에, `+=`를 할 때마다 기존 문자열을 새로 복사한다. 그래서 질의가 `M`개일 때 누적 복사 비용이 커져 최악에는 `O(M^2)`에 가까워질 수 있다.

또한 아래 출력 방식도 문제가 있었다.

```python
print(" ".join(answer))
```

`answer`가 문자열이면 숫자 단위가 아니라 문자 단위로 공백이 들어간다. 예를 들어 어떤 숫자의 개수가 `10`이면 `10`이 아니라 `1 0`처럼 출력될 수 있다.

## 개선 후 확인한 점

개선한 코드에서는 `answer`를 리스트로 만들고, 각 결과를 리스트에 저장한 뒤 마지막에 `join`한다.

```python
answer = [None] * m
for i, target in enumerate(m_list):
    answer[i] = str(counter[target]) if counter[target] else '0'

print(" ".join(answer))
```

이제 결과를 모으는 과정은 질의마다 한 번씩만 처리되므로, 전체 시간 복잡도는 평균적으로 다음과 같다.

```text
Counter 생성: O(N)
질의 처리: O(M)
최종 출력 생성: O(M)
전체: O(N + M)
```

직접 확인한 테스트:

- 문제 예제
- 개수가 `10`처럼 두 자리 이상 나오는 경우
- 카드가 1개이고 질의 숫자가 없는 값인 최소 크기 근처 경우

위 테스트들은 모두 기대한 출력과 일치했다.

## 더 다듬을 수 있는 점

`Counter`는 없는 키를 조회하면 자동으로 `0`을 반환한다. 따라서 아래 조건문은 없어도 된다.

```python
answer[i] = str(counter[target]) if counter[target] else '0'
```

조금 더 단순하게 쓰면 다음과 같다.

```python
answer[i] = str(counter[target])
```

또한 파일 맨 위 주석의 `같은 수는 없음`은 이 문제 조건과 반대다. 이 문제에서는 같은 숫자 카드가 여러 장 있을 수 있으므로, 주석은 지우거나 `같은 수가 있을 수 있음`으로 바꾸는 것이 좋다.

## 다음 문제에 적용할 교훈

출력을 여러 개 모아야 할 때는 문자열에 계속 더하지 말고, 리스트에 담은 뒤 마지막에 `join`하는 습관을 들이면 시간 복잡도와 출력 형식 실수를 줄일 수 있다.
