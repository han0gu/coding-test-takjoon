# Codex 피드백

## 현재 코드가 틀리는 이유

현재 `main.py`는 두 방정식을 각각 확인하지 않고, 두 방정식을 더한 식 하나만 확인하고 있습니다.

문제에서 만족해야 하는 조건은 아래 두 개입니다.

```text
ax + by = c
dx + ey = f
```

그런데 현재 코드는 두 식을 더해서 만든 아래 조건만 확인합니다.

```text
(a + d)x + (b + e)y = c + f
```

이 조건은 두 식을 모두 만족하면 당연히 만족합니다.
하지만 반대로, 이 조건을 만족한다고 해서 두 식을 각각 만족한다는 뜻은 아닙니다.

## 왜 더한 식만으로는 부족한가?

두 식을 각각 만족한다는 것은 다음 두 조건이 동시에 맞아야 한다는 뜻입니다.

- 첫 번째 식의 왼쪽 값이 정확히 `c`와 같다.
- 두 번째 식의 왼쪽 값이 정확히 `f`와 같다.

하지만 두 식을 더해서 확인하면, 첫 번째 식에서 생긴 오차와 두 번째 식에서 생긴 오차가 서로 상쇄될 수 있습니다.

예를 들어 첫 번째 식은 정답보다 `+2200`만큼 크고, 두 번째 식은 정답보다 `-2200`만큼 작다고 생각해봅시다.
그러면 각각의 식은 둘 다 틀렸지만, 두 식을 더한 값은 우연히 맞아질 수 있습니다.

즉 관계는 아래처럼 한쪽 방향만 성립합니다.

```text
두 식을 각각 만족한다 -> 더한 식도 만족한다
더한 식을 만족한다 -> 두 식을 각각 만족한다  (성립하지 않음)
```

## 예제 1로 확인하기

예제 1은 다음 입력입니다.

```text
1 3 -1 4 1 7
```

정답은 다음과 같습니다.

```text
2 -1
```

하지만 현재 코드는 먼저 찾은 값인 `-798 999`를 출력합니다.

이 값을 첫 번째 식에 넣어보면 다음과 같습니다.

```text
x + 3y = -798 + 3 * 999 = 2199
```

첫 번째 식의 오른쪽 값은 `-1`이어야 하므로, 이 값은 첫 번째 식을 만족하지 않습니다.
그런데도 두 식을 더한 조건만 보면 통과해버립니다.

## 고치는 방향

완전탐색으로 `x`, `y` 후보를 모두 확인하는 방향은 문제 조건에 잘 맞습니다.
다만 후보를 찾았을 때는 더한 식 하나가 아니라, 원래 문제에 있는 두 식을 모두 만족하는지 확인해야 합니다.

확인해야 할 질문은 하나입니다.

```text
이 x, y가 첫 번째 식도 만족하고, 두 번째 식도 만족하는가?
```

이 조건을 기준으로 `if`문을 바꾸면 현재 풀이 흐름을 그대로 살릴 수 있습니다.

## 다음 문제에 적용할 점

여러 조건을 동시에 만족해야 하는 문제에서는 조건을 합치거나 변형했을 때 정보가 사라질 수 있습니다.
특히 방정식을 더하는 것은 필요한 조건을 하나 만들 뿐, 원래 두 조건을 완전히 대체하지는 못한다는 점을 기억하면 좋습니다.
