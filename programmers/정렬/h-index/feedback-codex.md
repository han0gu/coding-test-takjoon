# H-Index 피드백

## 문제 조건 요약

- 논문별 인용 횟수 배열 `citations`가 주어진다.
- `h`번 이상 인용된 논문이 `h`편 이상이고, 나머지 논문이 `h`번 이하 인용되었다면 `h`가 H-Index 후보가 된다.
- 가능한 `h` 중 최댓값을 반환해야 한다.
- 논문 수는 최대 `1,000`편, 인용 횟수는 최대 `10,000`회다.

## 현재 풀이

현재 코드는 인용 횟수를 내림차순으로 정렬한 뒤, 가능한 `h` 값을 큰 값부터 확인한다.

```python
def solution(citations):
    n = len(citations)
    citations.sort(reverse=True)

    for i in range(n, 0, -1):
        if citations[i-1] >= i:
            return i

    return 0
```

핵심은 `citations[i-1] >= i`다. 내림차순 정렬 상태에서 `i`번째 논문이 `i`번 이상 인용되었다면, 그 앞의 논문들은 모두 `i`번 이상 인용되었다고 볼 수 있다.

또한 `i`를 큰 값부터 줄여가며 확인하므로, 처음 조건을 만족하는 `i`가 곧 가능한 H-Index의 최댓값이다.

## Gemini 리뷰 요약

Gemini는 현재 코드가 오답이라고 지적한 것이 아니라, 다음 이유로 `enumerate`를 쓰는 방식이 더 읽기 좋다고 제안했다.

- `range(n, 0, -1)`와 `citations[i-1]` 조합은 직관적으로 읽기 어렵다.
- 내림차순 정렬된 배열을 앞에서부터 보면서, 인용 횟수가 `index + 1`보다 작아지는 지점을 찾으면 된다.
- 이렇게 쓰면 `n` 변수를 따로 둘 필요가 없다.

제안 코드는 다음과 같다.

```python
def solution(citations):
    citations.sort(reverse=True)

    for i, citation in enumerate(citations):
        if citation < i + 1:
            return i

    return len(citations)
```

단, Gemini의 설명 중 "사용하지 않는 변수 `n`"이라는 표현은 정확하지 않다. 현재 코드에서 `n`은 `range(n, 0, -1)`에 사용되고 있다. 더 정확히는 "`enumerate` 방식으로 바꾸면 `n` 변수를 따로 만들 필요가 없다"는 뜻이다.

## 두 풀이가 같은 이유

예를 들어 내림차순 정렬 후 배열이 다음과 같다고 하자.

```python
[6, 5, 3, 1, 0]
```

`enumerate` 방식에서는 다음 조건을 앞에서부터 확인한다.

```python
if citation < i + 1:
```

`i = 0`이면 논문 1편이 1번 이상 인용되었는지 본다.
`i = 1`이면 논문 2편이 각각 2번 이상 인용되었는지 본다.
`i = 2`이면 논문 3편이 각각 3번 이상 인용되었는지 본다.

처음으로 `citation < i + 1`이 되는 순간, `i + 1`편의 논문이 `i + 1`번 이상 인용되었다고 말할 수 없다. 따라서 바로 직전 값인 `i`가 H-Index가 된다.

현재 코드의 역순 확인 방식은 반대로 가능한 `h`를 큰 값부터 직접 검사한다.

```python
for i in range(n, 0, -1):
    if citations[i-1] >= i:
        return i
```

두 방식은 확인 방향만 다를 뿐, 모두 정렬된 배열에서 "몇 편의 논문이 그 편수 이상 인용되었는가"를 판단한다.

## 검증한 내용

현재 코드, Gemini 제안 코드, 단순 oracle을 같은 입력에 대해 비교했다.

```text
예제 케이스
단일 논문 케이스
전체 0회 인용 케이스
전체 고인용 케이스
경계값 케이스
랜덤 케이스 500개
```

위 케이스에서 세 방식은 모두 같은 결과를 냈다.

따라서 Gemini 리뷰는 문제를 잘못 이해한 오답 지적이라기보다, 같은 정답 로직을 더 Pythonic하게 표현할 수 있다는 개선 제안에 가깝다.

## 정리

- 현재 코드는 정답 방향이다.
- Gemini가 제안한 `enumerate` 풀이도 정답 방향이다.
- 차이는 정답성보다 가독성에 있다.
- 학습 관점에서는 현재 코드처럼 `h` 후보를 큰 값부터 직접 확인하는 방식도 이해하기 좋다.
- Pythonic한 표현을 연습하려면 Gemini 방식처럼 앞에서부터 실패 지점을 찾는 풀이도 익혀둘 만하다.
