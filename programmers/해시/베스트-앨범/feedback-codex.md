# 베스트 앨범 피드백

## 문제 조건 요약

- 장르별 총 재생 수가 큰 장르를 먼저 수록한다.
- 같은 장르 안에서는 재생 수가 큰 노래를 먼저 수록한다.
- 같은 장르 안에서 재생 수가 같으면 고유 번호가 낮은 노래를 먼저 수록한다.
- 장르마다 최대 2곡까지만 수록한다.

## `TypedDict` 제거 방향

처음에는 `TypedDict`로 장르별 정보를 표현하려고 했다. 장르별로 `total`과 `songs`를 묶어서 관리하려는 의도는 좋았다.

다만 `TypedDict`는 런타임에서 기본값을 만들어주는 도구가 아니라, dict의 구조를 설명하는 타입 힌트에 가깝다.

```python
class MyType(TypedDict):
    total = 0
    song = []
```

이런 방식으로는 `{"total": 0, "songs": []}` 형태의 기본값이 자동 생성되지 않는다.

현재처럼 `TypedDict`를 제거하고 `defaultdict`의 기본값 생성 함수를 명시한 방식이 이 문제에는 더 자연스럽다.

```python
d = defaultdict(lambda: {'total': 0, 'songs': []})
```

이렇게 쓰면 새 장르가 처음 등장했을 때마다 독립적인 dict와 독립적인 리스트가 생성된다.

## 현재 풀이의 좋은 점

장르별 총 재생 수와 노래 목록을 한 번에 관리하는 구조가 좋다.

```python
d[g]['songs'].append((i, plays[i]))
d[g]['total'] += plays[i]
```

이렇게 하면 장르 순서를 정할 때는 `total`을 사용하고, 장르 안에서 곡을 고를 때는 `songs`를 사용할 수 있다.

장르 정렬도 문제 조건과 잘 맞는다.

```python
sorted_keys = sorted(d, key=lambda x: -d[x]['total'])
```

장르별 총 재생 수는 모두 다르다는 조건이 있으므로, 총 재생 수 내림차순만 고려해도 충분하다.

## 정렬 기준 개선

현재 장르 안의 곡 정렬은 다음과 같다.

```python
songs.sort(key=lambda x: -x[1])
```

`songs`에는 `(고유 번호, 재생 수)`가 들어 있다. 따라서 `x[1]`은 재생 수이고, `-x[1]`은 재생 수 내림차순 정렬을 의미한다.

이 코드는 실제로 동작한다. Python 정렬은 stable sort이므로, 재생 수가 같은 경우 기존 순서가 유지된다. 그리고 `songs`는 원래 인덱스 오름차순으로 추가되므로, 결과적으로 고유 번호가 낮은 노래가 먼저 온다.

다만 문제 조건을 코드에 더 직접적으로 드러내려면 정렬 기준을 명시하는 편이 좋다.

```python
songs.sort(key=lambda x: (-x[1], x[0]))
```

이 의미는 다음과 같다.

1. `-x[1]`: 재생 수 내림차순
2. `x[0]`: 고유 번호 오름차순

현재 코드도 맞지만, 위처럼 쓰면 “재생 수가 같으면 고유 번호가 낮은 노래 먼저”라는 조건이 코드에 바로 보인다.

## 변수명

`sorted_keys`는 동작상 문제는 없지만, 여기서 정렬하는 것은 단순 key가 아니라 장르다.

```python
sorted_keys = sorted(d, key=lambda x: -d[x]['total'])
```

다음처럼 쓰면 의미가 더 분명하다.

```python
sorted_genres = sorted(d, key=lambda genre: -d[genre]['total'])
```

마찬가지로 `d`도 짧은 변수명이라 코딩 테스트에서는 괜찮지만, 여유가 있다면 `genre_info`처럼 쓰면 장르별 정보를 담는 dict라는 점이 더 잘 드러난다.

## 정리

현재 풀이의 전체 방향은 좋다.

- `defaultdict(lambda: {'total': 0, 'songs': []})`로 장르별 정보를 모으는 방식이 적절하다.
- 장르별 총 재생 수를 기준으로 장르를 정렬하는 흐름도 맞다.
- 장르별로 최대 2곡만 뽑는 처리도 문제 조건과 맞다.

추가로 다듬는다면 다음 두 가지가 핵심이다.

1. 장르 안 곡 정렬을 `(-재생 수, 고유 번호)` 형태로 명시한다.
2. `sorted_keys`, `d` 같은 변수명을 조금 더 의미 있게 바꾼다.
