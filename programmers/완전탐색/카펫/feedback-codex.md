# 카펫 피드백

## 문제 조건 요약

- 갈색 칸은 카펫의 테두리 1줄을 이룬다.
- 노란색 칸은 갈색 테두리 안쪽 영역을 이룬다.
- 전체 카펫의 가로 길이는 세로 길이 이상이다.
- `brown`은 8 이상 5,000 이하, `yellow`는 1 이상 2,000,000 이하이다.

## 기존 풀이의 좋은 점

기존 풀이는 전체 넓이를 먼저 구한 뒤, 가능한 가로 길이를 후보로 잡아 약수 쌍을 찾았다.

```python
total = brown + yellow
```

카펫의 전체 칸 수는 `가로 * 세로`이므로, `total`의 약수 쌍을 찾는 접근은 문제에 잘 맞는다.

또한 갈색 테두리 개수를 아래 식으로 검증한 것도 수학적으로 맞다.

```python
brown == (width * 2 + height * 2 - 4)
```

가로 2줄, 세로 2줄을 더하면 네 꼭짓점이 중복되므로 `-4`를 해주는 식이다.

## 개선한 점

기존 식도 맞지만, 이 문제는 그림상 노란색 영역이 더 직관적인 조건이다. 전체 카펫에서 테두리 한 줄을 제거하면 내부 영역의 크기는 다음과 같다.

```python
(width - 2) * (height - 2)
```

이 값이 `yellow`와 같으면 정답이다.

```python
if (width - 2) * (height - 2) == yellow:
    return [width, height]
```

이 조건은 “갈색 테두리 안쪽이 노란색 영역”이라는 문제 설명과 바로 연결된다.

## 탐색 범위

가로는 세로 이상이므로, 약수 쌍을 찾을 때 세로 후보만 확인해도 된다.

```python
for height in range(3, int(total ** 0.5) + 1):
```

세로가 `sqrt(total)`보다 커지면 가로보다 커지는 영역으로 넘어가므로, 그 이후는 중복 확인이다.

또한 `yellow >= 1`이므로 내부 영역이 최소 1칸은 있어야 한다. 따라서 전체 카펫의 세로는 최소 3이다.

```text
height = 3
내부 높이 = height - 2 = 1
```

그래서 세로 후보를 3부터 시작할 수 있다.

## 정리

이 문제의 핵심은 전체 넓이의 약수 쌍을 찾는 것이다.

기존 풀이처럼 갈색 테두리 개수로 검증해도 맞지만, 노란색 내부 영역 조건으로 검증하면 문제 그림과 더 직접적으로 연결된다.

정리하면 다음 흐름이 가장 자연스럽다.

1. `total = brown + yellow`를 구한다.
2. 가능한 세로 후보를 3부터 `sqrt(total)`까지 확인한다.
3. `total % height == 0`이면 가로를 계산한다.
4. `(width - 2) * (height - 2) == yellow`이면 `[width, height]`를 반환한다.
