# 멀리 뛰기 피드백

## 문제 조건 요약

- 한 번에 1칸 또는 2칸을 뛸 수 있다.
- `n`칸 끝에 도달하는 방법의 수를 구해야 한다.
- 결과는 `1234567`로 나눈 나머지를 반환한다.
- `n`은 1 이상 2,000 이하이다.

## DP 접근이 맞는 이유

이 문제는 dynamic programming으로 접근하는 것이 맞다.

`n`칸에 도착하는 마지막 움직임은 두 가지뿐이다.

1. `n - 1`칸에서 1칸 뛰어서 도착한다.
2. `n - 2`칸에서 2칸 뛰어서 도착한다.

따라서 `n`칸에 도착하는 방법의 수는 다음과 같이 표현할 수 있다.

```text
dp[n] = dp[n - 1] + dp[n - 2]
```

이전 상태의 답을 이용해서 현재 답을 구하므로 DP 구조가 맞다. 형태는 피보나치 수열과 같다.

## 현재 풀이의 좋은 점

현재 코드는 `dp` 배열을 직접 만들지 않고, 직전 두 값만 변수로 들고 간다.

```python
a = 1
b = 2
```

여기서 `a`는 `dp[i - 2]`, `b`는 `dp[i - 1]` 역할을 한다. 다음 값을 구한 뒤 두 변수를 한 칸씩 밀어주는 방식이다.

```python
tmp = b
b += a
a = tmp
```

따라서 전체 배열을 저장하지 않아도 되므로 공간 복잡도는 `O(1)`이다. DP 배열을 생략한 공간 최적화 DP라고 볼 수 있다.

## 개선할 점

### 1. 모듈러 연산은 반복 중에 적용하는 편이 좋다

현재 코드는 마지막에만 나머지를 구한다.

```python
return b % 1234567
```

`n <= 2000`이라 Python에서는 큰 문제가 되지 않지만, 일반적으로는 반복 중에도 값을 `1234567` 이하로 유지하는 편이 좋다. 숫자가 불필요하게 커지지 않고, 문제에서 요구하는 조건도 계산 과정에 바로 반영된다.

```python
b = (a + b) % 1234567
```

모듈러 덧셈은 다음 성질 때문에 중간에 적용해도 결과가 같다.

```text
(x + y) % m = ((x % m) + (y % m)) % m
```

### 2. 변수 갱신은 tuple assignment로 더 간결하게 쓸 수 있다

현재처럼 `tmp`를 사용하는 방식도 맞다.

```python
tmp = b
b += a
a = tmp
```

다만 Python에서는 두 값을 동시에 갱신할 수 있다.

```python
a, b = b, (a + b) % 1234567
```

이렇게 쓰면 `a`와 `b`가 각각 다음 상태로 이동한다는 점이 더 짧게 드러난다.

## 정리

현재 풀이는 DP 접근이 맞고, 정답성도 좋다. 특히 배열을 만들지 않고 직전 두 값만 유지한 점은 좋은 공간 최적화다.

다음 단계에서 다듬는다면 아래 두 가지를 반영하면 된다.

1. 반복문 안에서 `% 1234567`을 적용한다.
2. `tmp` 대신 `a, b = b, ...` 형태의 동시 할당을 사용한다.

핵심 점화식은 끝까지 다음 하나로 기억하면 된다.

```text
현재 칸에 오는 방법 = 이전 칸에서 오는 방법 + 전전 칸에서 오는 방법
```
