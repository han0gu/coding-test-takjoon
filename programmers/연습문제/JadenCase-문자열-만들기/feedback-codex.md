# JadenCase 문자열 만들기 피드백

## 문제 조건 요약

- 문자열 `s`는 길이 1 이상 200 이하이다.
- `s`는 알파벳, 숫자, 공백 문자로만 이루어진다.
- 숫자는 단어의 첫 문자로만 나온다.
- 공백 문자가 연속해서 나올 수 있으므로, 기존 공백 개수를 보존해야 한다.

## 현재 풀이의 좋은 점

현재 풀이에서 가장 중요한 선택은 `split(' ')`을 사용한 점이다.

```python
ls = s.split(' ')
```

이 문제에서는 일반적인 `split()`을 쓰면 안 된다. `split()`은 연속 공백을 하나의 구분자로 처리하고, 앞뒤 공백도 제거해버린다. 반면 `split(' ')`은 공백 하나하나를 기준으로 나누기 때문에 연속 공백 사이의 빈 문자열도 유지된다.

예를 들어:

```python
'a  b'.split(' ')
# ['a', '', 'b']
```

이 빈 문자열 덕분에 다시 `' '.join(...)` 했을 때 원래 공백 개수를 복원할 수 있다.

## 아쉬운 점

현재 코드는 빈 문자열을 처리하기 위해 임시 문자 `@`를 넣었다가 마지막에 제거한다.

```python
if not s:
    answer.append('@')
    continue

return ' '.join(answer).replace('@', '')
```

문제 조건상 입력에는 알파벳, 숫자, 공백만 나오므로 `@`가 입력과 충돌하지는 않는다. 그래서 현재 풀이가 채점에서 틀릴 가능성은 낮다. 다만 임시 문자를 끼워 넣었다가 제거하는 방식은 문제를 조금 돌아서 푸는 구조다.

사실 빈 문자열은 그대로 `''`로 두면 된다. `join`은 원소 사이에만 공백을 넣기 때문에, 빈 문자열을 그대로 유지해도 기존 공백 개수가 보존된다.

```python
' '.join(['a', '', 'b'])
# 'a  b'
```

즉, 별도의 placeholder 없이도 공백 보존이 가능하다.

## 더 단순한 표현

각 단어는 첫 글자만 대문자, 나머지는 소문자가 되어야 한다. 이 동작은 `capitalize()`와 잘 맞는다.

```python
def solution(s):
    return ' '.join(word.capitalize() for word in s.split(' '))
```

`capitalize()`는 빈 문자열에 대해서도 오류 없이 빈 문자열을 반환한다. 따라서 연속 공백이 있는 경우에도 그대로 처리할 수 있다.

예를 들어:

```python
'  hello   world  '.split(' ')
# ['', '', 'hello', '', '', 'world', '', '']
```

각 원소에 `capitalize()`를 적용해도 빈 문자열은 그대로 남고, 마지막에 `' '.join(...)`을 하면 원래 공백 구조가 유지된다.

## 변수명

현재 반복문에서 함수 인자 `s`와 같은 이름을 다시 사용하고 있다.

```python
for s in ls:
```

동작상 문제는 없지만, `word` 같은 이름을 쓰면 “전체 문자열”과 “단어 하나”가 더 명확히 구분된다.

## 정리

현재 풀이는 핵심 조건인 연속 공백 보존을 잘 의식하고 있어서 방향이 좋다. 다만 `@` 같은 임시 문자를 사용하지 않고, `split(' ')`으로 생긴 빈 문자열을 그대로 유지한 채 `join`하는 방식이 더 자연스럽다.

이 문제의 핵심은 다음 두 가지다.

1. `split()`이 아니라 `split(' ')`을 사용해야 공백 개수를 보존할 수 있다.
2. 빈 문자열은 별도로 바꾸지 말고 그대로 두면 `join`이 공백을 자연스럽게 복원한다.
