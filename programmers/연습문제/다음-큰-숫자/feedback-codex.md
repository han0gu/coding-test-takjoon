# 다음 큰 숫자 피드백

## 문제 조건 요약

- `n`보다 큰 자연수 중에서 조건을 만족하는 가장 작은 수를 찾는다.
- 조건은 `n`과 다음 숫자를 2진수로 바꿨을 때 `1`의 개수가 같아야 한다.
- `n`은 1 이상 1,000,000 이하이다.

## 현재 풀이의 좋은 점

현재 풀이는 `n + 1`부터 하나씩 증가시키면서 2진수의 `1` 개수를 비교한다.

```python
for i in range(n + 1, n * 4):
```

이 접근은 문제 정의와 직접적으로 맞닿아 있다. “n보다 큰 수 중 가장 작은 수”를 찾는 문제이므로, 작은 후보부터 순서대로 확인하다가 처음 조건을 만족하는 값을 반환하면 된다.

제한도 `n <= 1,000,000`이라서 이 방식은 충분히 통과 가능한 범위다. 실제로 샘플과 경계에 가까운 값에서도 빠르게 동작한다.

## 개선할 점

### 1. `bin()` 결과에 `str()`은 필요 없다

`bin(n)`은 이미 문자열을 반환한다.

```python
b = str(bin(n)[2:])
```

위 코드는 아래처럼 줄일 수 있다.

```python
b = bin(n)[2:]
```

다만 이 문제에서는 2진수 문자열 자체를 따로 저장하지 않아도 된다.

### 2. `1`의 개수는 `count()`로 셀 수 있다

현재 코드는 리스트를 만든 뒤 길이를 구한다.

```python
len([c for c in b if c == '1'])
```

이 방식도 맞지만, `1`의 개수를 세는 목적이라면 문자열의 `count()`가 더 직접적이다.

```python
bin(n).count('1')
```

리스트를 새로 만들지 않기 때문에 메모리도 조금 덜 쓰고, 코드 의도도 더 선명하다.

### 3. 다시 `int(b_i, 2)`로 바꿀 필요가 없다

현재 코드는 후보 숫자 `i`를 2진수 문자열로 바꿨다가, 조건을 만족하면 다시 정수로 변환한다.

```python
return int(b_i, 2)
```

하지만 `b_i`는 애초에 `i`를 2진수 문자열로 바꾼 값이다. 따라서 다시 정수로 바꾸면 원래 `i`와 같다.

```python
return i
```

로 충분하다.

### 4. `n * 4` 상한은 설명하기 애매하다

현재 풀이의 가장 아쉬운 부분은 탐색 범위의 끝을 `n * 4`로 잡은 점이다.

```python
for i in range(n + 1, n * 4):
```

실제로는 이 문제 제한 안에서 통과할 가능성이 높지만, 코드만 보고 “왜 4배까지만 보면 항상 답이 있는가?”를 설명하기 어렵다.

조건을 만족하는 다음 숫자는 반드시 존재하므로, 더 자연스러운 구조는 답을 찾을 때까지 증가시키는 것이다.

```python
target = bin(n).count('1')
candidate = n + 1

while bin(candidate).count('1') != target:
    candidate += 1

return candidate
```

이렇게 쓰면 불필요한 상한을 정하지 않아도 되고, 문제 정의와 코드가 더 잘 맞는다.

## 정리

현재 풀이는 문제를 잘 이해한 직접 탐색 방식이다. `n + 1`부터 확인하므로 처음 발견한 값이 곧 정답이라는 점도 맞다.

다만 Python 표현을 더 단순하게 만들 수 있다.

- `str(bin(...))`는 불필요하다.
- `1`의 개수는 `bin(x).count('1')`로 바로 셀 수 있다.
- 조건을 만족한 후보는 `int(b_i, 2)`가 아니라 `i`를 그대로 반환하면 된다.
- `n * 4` 같은 임의 상한보다는 `while`로 답을 찾을 때까지 탐색하는 편이 더 자연스럽다.
