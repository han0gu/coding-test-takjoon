# 귤 고르기 피드백

## 문제 조건 요약

- `k`개의 귤을 골라 상자에 담아야 한다.
- 고른 귤들의 크기 종류 수를 최소화해야 한다.
- `1 <= k <= tangerine의 길이 <= 100,000`이다.
- 귤의 크기 값은 최대 10,000,000까지 가능하다.

## 기존 풀이의 좋은 점

크기별 개수를 세기 위해 `Counter`를 사용한 점이 좋다.

```python
c = Counter(tangerine)
```

이 문제는 귤의 실제 크기 값보다, 각 크기가 몇 개씩 있는지가 중요하다. 따라서 빈도수를 먼저 구하는 접근이 문제에 잘 맞는다.

또한 개수가 많은 크기부터 선택하는 방향도 맞다. 종류 수를 최소화하려면 한 종류를 선택했을 때 최대한 많은 귤을 확보해야 하므로, 빈도가 높은 크기부터 고르는 것이 유리하다.

## 그리디가 성립하는 이유

목표는 `k`개를 채우는 데 필요한 크기 종류 수를 최소화하는 것이다.

같은 종류 수를 선택한다고 가정하면, 더 많이 등장한 크기를 고르는 쪽이 항상 더 많은 귤을 확보한다. 따라서 적게 등장한 크기를 먼저 고르는 선택은 많이 등장한 크기를 먼저 고르는 선택보다 유리할 수 없다.

그래서 크기별 개수를 내림차순으로 정렬한 뒤, 큰 빈도부터 누적하면 최소 종류 수를 구할 수 있다.

## 개선한 점

기존 풀이에서는 `(크기, 개수)` 튜플 리스트를 직접 만들었다.

```python
ls = []
for key in c:
    ls.append((key, c[key]))
ls.sort(key=lambda x: -x[1])
```

하지만 이 문제에서는 크기 값 자체는 더 이상 필요하지 않다. 필요한 것은 각 크기의 개수뿐이다.

```python
counts = sorted(Counter(tangerine).values(), reverse=True)
```

이렇게 쓰면 “빈도만 내림차순으로 확인한다”는 의도가 더 직접적으로 드러난다.

## 반복 구조

기존 풀이의 `while True`도 문제 조건상 동작은 하지만, 종료 조건이 코드 겉으로 잘 드러나지 않는다.

빈도 리스트를 순회하면서 `k`에서 선택한 개수를 빼고, `k <= 0`이 되는 순간 선택한 종류 수를 반환하면 흐름이 더 명확하다.

```python
answer = 0
for count in counts:
    k -= count
    answer += 1
    if k <= 0:
        return answer
```

## Gemini 리뷰 반영

Gemini는 위 반복 구조에서 `answer`를 직접 1씩 증가시키는 대신 `enumerate(counts, 1)`을 사용할 수 있다고 피드백했다.

```python
for answer, count in enumerate(counts, 1):
    k -= count
    if k <= 0:
        return answer
```

`enumerate`의 두 번째 인자는 시작 인덱스를 뜻한다. 따라서 `enumerate(counts, 1)`에서 `answer`는 1, 2, 3, ... 순서로 증가하고, 현재까지 선택한 귤 크기 종류 수를 바로 나타낸다.

이렇게 쓰면 별도의 `answer = 0` 초기화와 `answer += 1` 증가 코드가 사라져 반복문의 역할이 더 간결해진다.

## 정리

이 문제의 핵심은 다음과 같다.

1. `Counter`로 크기별 개수를 센다.
2. 개수만 내림차순으로 정렬한다.
3. 많이 등장한 크기부터 선택하면서 `k`개 이상이 되는 순간의 선택 개수를 반환한다.

기존 풀이의 알고리즘 방향은 맞았다. 개선 포인트는 크기 값이 필요 없다는 점을 코드에 반영하고, `while True` 대신 `for` 순회로 종료 조건을 더 명확하게 만드는 것이다.
