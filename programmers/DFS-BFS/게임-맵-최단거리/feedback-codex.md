# 게임 맵 최단거리 피드백

## 문제 조건 요약

- `maps`는 최대 `100 x 100` 크기의 격자이며, `1`은 이동 가능 칸, `0`은 벽이다.
- 시작점은 `(0, 0)`, 도착점은 `(n - 1, m - 1)`이다.
- 도착점까지 이동할 수 있으면 지나간 칸의 개수를 반환하고, 불가능하면 `-1`을 반환한다.

## 현재 DFS 풀이의 방향

현재 풀이는 `stack`에 `(row, col, acc)`를 넣고 탐색하므로 형태 자체는 DFS에 가깝다.

```python
stack = [(0, 0, 1)]
visited = [[False] * m for _ in range(n)]
```

`acc`에 현재까지 지나온 칸 수를 함께 들고 다니고, 도착점에 도달하면 `answer`를 갱신하려는 접근도 의도는 자연스럽다.

문제는 `visited`를 전역 방문 처리처럼 사용한다는 점이다.

## 핵심 문제: DFS의 전역 visited는 최단거리를 보장하지 않는다

DFS는 한 경로를 깊게 따라 들어간다. 그래서 어떤 칸을 처음 방문했다고 해서, 그 칸에 최단거리로 도착했다는 뜻은 아니다.

현재 코드는 어떤 칸을 한 번 방문하면 이후 다른 경로에서 그 칸으로 다시 들어오지 못하게 막는다.

```python
if visited[row][col]:
    continue

visited[row][col] = True
```

이 구조에서는 더 긴 경로가 먼저 어떤 칸이나 도착점에 도달하면, 나중에 발견될 수 있는 더 짧은 경로가 `visited` 때문에 잘릴 수 있다.

예를 들어 아래처럼 모든 칸이 이동 가능한 `2 x 3` 격자를 생각해볼 수 있다.

```python
maps = [
    [1, 1, 1],
    [1, 1, 1],
]
```

이 경우 최단거리는 다음처럼 `4`다.

```text
(0, 0) -> (0, 1) -> (0, 2) -> (1, 2)
```

하지만 현재 DFS 구현은 스택의 처리 순서 때문에 더 깊은 우회 경로를 먼저 따라갈 수 있고, 도착점 `(1, 2)`를 거리 `6`으로 먼저 방문한다. 이후 거리 `4`로 도착점에 도달하는 후보가 스택에 남아 있어도, 도착점이 이미 `visited=True`라서 건너뛰게 된다.

즉, 도착점에서 `min(answer, acc)`를 하더라도 이미 짧은 경로 자체가 탐색에서 제거되기 때문에 정답을 보장할 수 없다.

## DFS로 풀려면 어떻게 달라져야 하나

DFS로도 이론상 풀 수는 있다. 다만 현재처럼 전역 `visited`를 유지하면 안 되고, 하나의 경로를 탐색한 뒤에는 방문 상태를 되돌리는 백트래킹이 필요하다.

개념적으로는 다음 흐름이 되어야 한다.

```python
visited[row][col] = True

for next_row, next_col in candidates:
    if not visited[next_row][next_col]:
        dfs(next_row, next_col, acc + 1)

visited[row][col] = False
```

이렇게 해야 "현재 경로에서 이미 지나간 칸"만 막고, 다른 경로에서는 같은 칸을 다시 사용할 수 있다.

또한 이미 찾은 답보다 현재 거리가 길거나 같다면 더 탐색하지 않는 가지치기를 넣을 수 있다.

```python
if answer != -1 and acc >= answer:
    return
```

다만 이 방식은 가능한 경로를 많이 탐색해야 하므로, 격자가 커질수록 경우의 수가 급격히 늘어난다. 문제의 최대 크기가 `100 x 100`이기 때문에 DFS 백트래킹으로 모든 후보 경로를 비교하는 방식은 실전 풀이로 적합하지 않다.

## 정리

현재 코드는 "스택을 사용한 DFS"라는 점에서는 방향이 맞지만, "최단거리 문제를 DFS로 처리하는 방식"으로는 정답성이 깨진다.

이 문제에서 중요한 구분은 다음이다.

- DFS + 전역 `visited`: 한 번 방문한 칸을 막기 때문에 더 짧은 경로를 놓칠 수 있다.
- DFS + 백트래킹 `visited`: 정답성은 만들 수 있지만 가능한 경로가 너무 많아 비효율적이다.
- 최단거리 탐색: 같은 비용으로 한 칸씩 이동하는 격자에서는 가까운 거리부터 확정하는 방식이 더 자연스럽다.

따라서 이 문제를 DFS로 이해하려면, "DFS는 가능한 경로를 깊게 확인하는 도구"이고 "최단거리를 보장하려면 방문 처리 방식이 별도로 정교해야 한다"는 점을 기억하면 된다.
