# Codex 피드백

## 문제 조건 요약

- `events` 배열에서 가장 자주 등장한 이벤트 ID `k`개를 반환한다.
- 빈도가 같으면 원래 `events` 배열에서 더 먼저 등장한 이벤트 ID를 우선한다.
- `events.length`는 최대 `100000`이고, 빈 배열이면 `k = 0`이다.

문제 파일은 PDF가 아니라 같은 디렉토리의 `top-k-fequent-events-with-order-preservation.png` 이미지로 확인했다.

## 현재 풀이 평가

현재 `main.py`의 풀이는 정답성 측면에서 큰 문제를 찾지 못했다.

```python
sorted_cnt_map = sorted(cnt_map, key=lambda x: cnt_map[x], reverse=True)
```

이 코드는 빈도 기준으로 내림차순 정렬한다. 여기서 중요한 점은 Python의 `dict`가 key 삽입 순서를 유지하고, `sorted`가 안정 정렬이라는 점이다.

따라서 `cnt_map`에는 이벤트 ID가 처음 등장한 순서대로 들어가고, 빈도가 같은 값끼리는 기존 순서가 유지된다. 결과적으로 문제의 동률 조건인 "먼저 등장한 이벤트 우선"도 만족한다.

## 개선하면 좋은 점

현재 코드는 Python의 정렬 특성을 잘 활용하고 있지만, 동률 처리 의도가 코드에 직접 드러나지는 않는다. 문제를 처음 읽는 사람이 보면 "빈도가 같을 때 먼저 등장한 값을 고르는 로직이 어디 있지?"라고 헷갈릴 수 있다.

그래서 첫 등장 위치를 따로 저장하고, 정렬 기준에 명시하면 더 읽기 쉬운 풀이가 된다.

```python
def getTopKFrequentEvents(events, k):
    count = {}
    first_index = {}

    for i, event in enumerate(events):
        if event not in count:
            count[event] = 0
            first_index[event] = i
        count[event] += 1

    result = sorted(count, key=lambda event: (-count[event], first_index[event]))
    return result[:k]
```

여기서 정렬 key는 tuple이다.

```python
key=lambda event: (-count[event], first_index[event])
```

Python은 tuple을 앞 원소부터 차례대로 비교한다.

1. `-count[event]`: 빈도가 높은 이벤트가 먼저 오도록 한다.
2. `first_index[event]`: 빈도가 같으면 먼저 등장한 이벤트가 먼저 오도록 한다.

tuple에는 3개 이상의 기준을 넣을 수도 있다. 예를 들어 `(-count[event], first_index[event], event)`처럼 쓰면 빈도, 첫 등장 위치, 이벤트 값 순서로 비교한다. 다만 이 문제에서는 앞의 두 기준만 있으면 충분하다.

## 복잡도

- 시간 복잡도: `O(n + D log D)`
  - `n`: 전체 이벤트 수
  - `D`: 서로 다른 이벤트 ID 개수
- 공간 복잡도: `O(D)`

제한이 `events.length <= 100000`이므로 충분히 통과 가능한 복잡도다.

## 다음 문제에 적용할 점

동률 조건이 있는 정렬 문제에서는 정렬 key에 조건을 직접 드러내면 좋다.

현재 풀이처럼 언어의 안정 정렬 특성을 활용하는 것도 가능하지만, 코딩 테스트에서는 `(-빈도, 첫_등장_위치)`처럼 기준을 명시하면 실수도 줄고 설명하기도 쉬워진다.
